# Skills System

> Procedural protocols attached to work stages, not roles.

## What Are Skills?

Skills fill the gap between habits (high-level behavioral patterns) and role specs (identity + responsibilities). A skill is a **step-by-step protocol** that ensures consistent quality for specific types of work, regardless of which role produces it.

**Key distinction**: Roles have identity and decision rights. Skills have procedures and checklists. Any role can invoke any skill.

## When to Use Skills

Skills are triggered by **work stage**, not by topic:

| Stage | Skill | Trigger |
|-------|-------|---------|
| Before any investigation or analysis | Investigation Framing | "Why is X?", "Analyze Y", root cause, comparison |
| Before presenting completed work | Pre-Ship Review | Work item completion, deliverable ready, "done" declaration |
| Before presenting high-stakes recommendations | Structured Challenge | Irreversible decisions, cross-org proposals, investment recommendations |

## How to Invoke

When the trigger condition is met, read the skill file and follow its protocol before proceeding. The skill's output format becomes part of your deliverable.

## Customization

Add project-specific skills by creating new `.skill.md` files in this directory and adding their triggers to `selector.md`.
