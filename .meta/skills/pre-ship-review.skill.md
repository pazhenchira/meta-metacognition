# Pre-Ship Review

> Chain-of-Verification before presenting completed work.

## Why This Skill Exists

The most trust-damaging failure: presenting "completed" work that the recipient immediately finds defective. Work was PRODUCED but never REVIEWED from the consumer's perspective. This skill enforces that review.

## Protocol: Chain-of-Verification

### Step 1 — Restate the Original Ask

What was requested? Quote or paraphrase the original request. This is the standard against which everything is checked.

### Step 2 — Decompose into Verifiable Claims

List every discrete claim, output element, or deliverable component. Each must be independently checkable.

Include:
- Content claims (is the analysis correct?)
- Structural claims (is it well-organized?)
- Completeness claims (are all items addressed?)
- Negative claims (no duplicate content, no broken references)

### Step 3 — Independently Verify Each Claim

For each claim, re-read the relevant artifact. Do NOT rely on memory from the production step.

**Critical**: "PASS — verified" is NOT valid evidence. Describe WHAT YOU SAW. "PASS — re-read lines 1-15, summary appears once, no duplicate" is valid.

### Step 4 — Check Completeness Against Original Ask

Map each element of the original ask to the deliverable. If any element is not addressed, the deliverable is incomplete.

### Step 5 — Consumer Perspective Check

Read the deliverable as the RECIPIENT would:
- What will they see first?
- Does the flow make sense from their perspective?
- Would they need to ask clarifying questions?
- Is it ready to use as-is?

### Step 6 — Artifact-Type Checklist

Apply the checklist matching the artifact type:

**Analysis**: MECE structure? Differential explained (not just one side)? Fact vs inference vs assumption distinguished? Synthesis, not data dump?

**Document**: Reader has context at every point? Executive summary standalone? Structure tells a story? Specific over abstract?

**Recommendation**: Decision framed (not just topic)? Tradeoffs named? Counter-indicator stated? Insight connected to specific action?

## Output Format

```
### Review Verdict
- **Claims checked**: [N total, N passed, N failed]
- **Completeness**: [all elements addressed / gaps listed]
- **Consumer ready**: [yes — ready to use / no — needs specific work]
- **Defects found**: [list each]
- **Defects fixed**: [list each with fix applied]
```
